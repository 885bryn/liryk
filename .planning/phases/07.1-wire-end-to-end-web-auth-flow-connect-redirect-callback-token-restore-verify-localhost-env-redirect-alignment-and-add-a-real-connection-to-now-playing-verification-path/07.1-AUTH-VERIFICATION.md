# Phase 07.1 Auth Verification Runbook

## 1) Required Environment Variables

Set these in your local `.env` before running the web shell:

- `SPOTIFY_CLIENT_ID`: Spotify app client id.
- `SPOTIFY_REDIRECT_URI`: callback URL used by Spotify auth (recommended `http://localhost:3000/callback`).
- `APP_BASE_URL`: web app base URL (recommended `http://localhost:3000`).
- `SPOTIFY_AUTH_SCOPES`: comma-separated scopes (for example `user-read-currently-playing,user-read-playback-state`).

Validation source of truth: `src/infra/config/env.ts`.

## 2) Spotify Dashboard Redirect URI Checklist

In Spotify Developer Dashboard for the local app:

1. Open app settings and locate Redirect URIs.
2. Add `SPOTIFY_REDIRECT_URI` exactly as configured locally.
3. Ensure protocol, host, port, and path match exactly (for example `http://localhost:3000/callback`).
4. Save settings before starting the flow.
5. If you use `127.0.0.1` in dashboard, match `127.0.0.1` in both `APP_BASE_URL` and `SPOTIFY_REDIRECT_URI`.

## 3) Automated Commands

Run from repository root:

```bash
npm test -- src/web/auth/web-auth-controller.test.ts src/web/auth/env-alignment.test.ts src/web/auth/local-verification-path.test.tsx
npm test -- src/web/app-shell.test.tsx src/app/auth-runtime.test.ts
npm run build
```

Expected result: all commands pass with zero test failures and a successful build.

## 4) Manual Connect-to-Now-Playing Walkthrough

1. Start the app with `npm run dev` and open the local shell URL.
2. Confirm startup shows `Checking Spotify connection...` in the connection pane.
3. Click `Connect Spotify` and complete Spotify auth in browser.
4. After callback return, confirm URL is cleaned (no `code`, `state`, or `error` query params remain in the address bar).
5. If callback/exchange fails, confirm shell shows reconnect-needed flow with `Reconnect Spotify` and reason-aware helper text.
6. If auth succeeds and no active playback, confirm shell shows `Connected - play a track on Spotify`.
7. Start playback in Spotify and confirm the app proceeds from connected-waiting state toward active lyrics playback behavior.
